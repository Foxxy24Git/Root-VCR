# Admin Print Voucher — Design Spec

**Date:** 2026-04-16  
**Status:** Approved  
**Scope:** Admin bisa cetak voucher tanpa potong saldo, tidak masuk laporan keuangan, tidak mengganggu flow reseller.

---

## 1. Goal

- Admin dapat generate/cetak voucher seperti reseller
- Tidak ada deduction saldo
- Tidak masuk laporan keuangan (revenue, analytics)
- Flow reseller tetap identik seperti sebelumnya
- UI responsive (desktop, tablet, mobile)

---

## 2. Approach

Tambah field `source` (enum `ADMIN` / `RESELLER`) ke tabel `vouchers`. Semua revenue queries filter `source = 'RESELLER'`. UI baru di admin dashboard untuk cetak voucher tanpa info saldo.

---

## 3. Database / Schema

### New Enum

```prisma
enum VoucherSource {
  ADMIN
  RESELLER
}
```

### Voucher Model — New Field

```prisma
model Voucher {
  // ... existing fields ...
  source  VoucherSource  @default(RESELLER)
}
```

**Default `RESELLER`** memastikan semua voucher lama otomatis dianggap dari reseller — tidak ada data corruption.

### Migration SQL (auto-generated by Prisma)

```sql
CREATE TYPE "VoucherSource" AS ENUM ('ADMIN', 'RESELLER');
ALTER TABLE "vouchers" ADD COLUMN "source" "VoucherSource" NOT NULL DEFAULT 'RESELLER';
```

## 4. API Changes

### `POST /api/vouchers/generate` — Minimal Changes

File: `src/app/api/vouchers/generate/route.ts`

**Change 1 — `price_charged = 0` untuk admin:**

```typescript
// BEFORE
let priceCharged = basePrice

// AFTER
let priceCharged = user.role === 'admin' ? 0 : basePrice
```

**Change 2 — set `source` saat create voucher:**

```typescript
tx.voucher.create({
  data: {
    code,
    user_id: user.id,
    profile_id: profileId,
    status: "unused",
    price_charged: priceCharged,
    source: user.role === 'admin' ? 'ADMIN' : 'RESELLER',  // NEW
  },
})
```

**Tidak ada API baru.** Admin dan reseller pakai endpoint yang sama. Role check di backend sudah handle semuanya.

**Response untuk admin:**
```json
{
  "vouchers": [...],
  "summary": {
    "total_cost": 0,
    "fee_percentage": 0,
    "mikrotik_synced": 3
  }
}
```

## 5. Revenue Queries — Filter by Source

Semua query yang aggregate `price_charged` atau count vouchers untuk laporan keuangan harus tambah filter `source: 'RESELLER'`.

### Files yang diupdate:

**`src/app/(dashboard)/admin/dashboard/page.tsx`**
```typescript
// revenueMTD
prisma.voucher.aggregate({
  where: { generated_at: { gte: startOfMonth }, source: 'RESELLER' },
  _sum: { price_charged: true }
})

// vouchersLast7Days (untuk revenue chart — pisahkan dari count total)
prisma.voucher.findMany({
  where: { generated_at: { gte: sevenDaysAgo }, source: 'RESELLER' },
  select: { generated_at: true, price_charged: true }
})
```

**`src/app/(dashboard)/admin/analytics/page.tsx`** — audit dan tambah filter yang sama pada query revenue.

**`src/app/(dashboard)/admin/revenue/RevenueClient.tsx`** — audit dan tambah filter jika ada query revenue.

> **Note:** `vouchersToday` (count total) boleh tetap tanpa filter — itu hanya hitungan operasional, bukan finansial.

## 6. UI — New Component

### `AdminPrintVoucherForm`

**File:** `src/app/(dashboard)/admin/dashboard/AdminPrintVoucherForm.tsx`

**Props:**
```typescript
interface AdminPrintVoucherFormProps {
  profiles: { id: string; name: string; duration_days: number; duration_hours: number }[]
}
```

**Behavior:**
- Dropdown pilih profile (semua `is_active = true`, tidak dibatasi assignment)
- Input jumlah voucher (1–100)
- Tombol "CETAK VOUCHER" — tidak ada info saldo, tidak disabled karena saldo
- POST ke `/api/vouchers/generate` dengan `{ profileId, quantity }`
- Sukses → buka `VoucherSuccessModal` (reuse existing)
- Error → tampil error message inline

**Perbedaan vs `GenerateVoucherForm` (reseller):**

| | Reseller | Admin |
|--|--|--|
| Props | `availableProfiles`, `currentBalance` | `profiles` saja |
| Tampil total biaya | Ya | Tidak |
| Disabled jika saldo kurang | Ya | Tidak |
| Source profile | Hanya yang di-assign | Semua aktif |
| Reuses VoucherSuccessModal | Ya | Ya |

## 7. UI — Admin Dashboard Layout

**File:** `src/app/(dashboard)/admin/dashboard/page.tsx`

Tambah fetch profiles aktif dan render `AdminPrintVoucherForm` sebagai section baru di bawah revenue chart + top resellers:

```tsx
// Fetch tambahan di server component
const activeProfiles = await prisma.profile.findMany({
  where: { is_active: true },
  orderBy: { price: 'asc' },
  select: { id: true, name: true, duration_days: true, duration_hours: true }
})

// Di JSX — section baru setelah existing grid
<div className="bg-white dark:bg-slate-800 rounded-2xl border border-slate-100 dark:border-slate-700 shadow-[0_1px_3px_rgba(0,0,0,0.08)] p-6">
  <h3 className="font-bold text-slate-900 dark:text-slate-100 mb-1">Cetak Voucher</h3>
  <p className="text-sm text-slate-500 dark:text-slate-400 mb-6">Generate voucher tanpa potong saldo</p>
  <AdminPrintVoucherForm profiles={activeProfiles} />
</div>
```

### Responsive Layout

- **Desktop (lg):** card di bawah chart grid, `max-w-md` atau full width tergantung konten
- **Tablet (md):** full width card
- **Mobile:** full width card, stacked
